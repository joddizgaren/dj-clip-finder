Bug 1 — The app crash (root cause of "30s timeout")
public/ was inside the ASAR archive but the server looked for it outside.
electron/main.ts sets ELECTRON_PUBLIC_PATH = process.resourcesPath/public/, which is a real folder outside the ASAR. But electron-builder.yml had public/ listed under files: — which puts it inside the compressed ASAR archive. So when the server called serveStatic(), fs.existsSync(distPath) returned false, it threw an error, and the server process died before ever calling listen() — causing the 30-second wait.
Fix: Removed dist/electron/public/**/* from files: and added it to extraResources: (both in electron-builder.yml and in the ebConfig object in build.electron.ts). Now public/ lands at resources/public/ exactly where main.ts expects it.

Bug 2 — The build failure (EBUSY)
Windows Defender scans .cjs files the moment they're created. The build script was jumping straight into electron-builder right after writing server.cjs, and antivirus still had a lock on it when electron-builder tried to copy it.
Fix: Added a 12-second cooldown pause between the esbuild step and the electron-builder step. Also increased the EBUSY retry attempts from 3 to 6 (with 10s between each), as a belt-and-suspenders backup.
