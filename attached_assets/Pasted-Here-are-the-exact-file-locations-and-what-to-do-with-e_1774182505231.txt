Here are the exact file locations and what to do with each:

1. script/build.electron.ts → Replace existing file at script/build.electron.ts
Key change: the Vite outDir is now path.resolve("dist/electron/public") instead of the plain string "dist/electron/public". Because Vite's root is set to the client/ subfolder, a relative string was being resolved from there, putting the frontend build into client/dist/electron/public/ instead of dist/electron/public/. Making it absolute with path.resolve() anchors it to the project root where everything else expects it.

2. electron/main.ts → Replace existing file at electron/main.ts
Key changes: getServerPath() and its dev fallback now reference server-bundle.js instead of server.cjs. A getLogPath() function and file logging were added to startServer() — all server stdout/stderr is written to a server.log file in the app's userData folder. The startup failure dialog now shows the path to that log file so errors are visible.

3. electron-builder.yml → Replace existing file at electron-builder.yml (root of project)
Key changes: npmRebuild: false was added (prevents electron-builder from trying to compile native modules, which was causing the paths[0] undefined error). afterPack: script/afterPack.cjs was added to hook into the packaging step. The public/ folder was removed from extraResources entirely because electron-builder's directory filter was silently failing every time. The signAndEditExecutable: false and signExts: [] settings were moved here from the old JS config object.

4. script/afterPack.cjs → Create new file at script/afterPack.cjs (same folder as build.electron.ts)
This is a brand new file. It's a plain Node.js CommonJS script that electron-builder calls automatically after packaging but before creating the installer. It does a simple recursive fs.copyFileSync of dist/electron/public/ into {appOutDir}/resources/public/. This bypasses electron-builder's own extraResources directory handling entirely, which was the root cause of the public folder never making it into the installer across many build attempts.
