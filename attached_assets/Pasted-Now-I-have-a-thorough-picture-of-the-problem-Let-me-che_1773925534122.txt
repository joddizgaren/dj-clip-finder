Here's a complete diagnosis of all the issues causing the "Server did not start within 30s" error:
Root Cause #1 — node_modules NOT bundled into the installer (Primary cause)
In script/build.electron.ts, the server is bundled with esbuild with all dependencies marked as external:
tsconst serverExternals = [...new Set([...ALWAYS_EXTERNAL, ...allPkgs])];
// Result: express, multer, better-sqlite3, pg, etc. are all external
This means server.cjs says require('express') at runtime, but node_modules is never copied into the installer. The files array in electron-builder.yml and in ebConfig only lists:
yaml- dist/electron/main.cjs
- dist/electron/preload.js
- dist/electron/public/**/*
- package.json
```
`node_modules/**/*` is missing. When the app runs on the user's machine, `server.cjs` crashes immediately trying to `require('express')` — and the 30-second timeout fires.

## Root Cause #2 — `better-sqlite3` is a native `.node` addon inside an ASAR archive

Even if `node_modules` were included, `better-sqlite3` contains a compiled binary (`*.node` file). Native addons **cannot be `require()`'d from inside an ASAR archive** — Electron will throw a "dlopen failed" error. You need `asarUnpack` to extract it.

## Root Cause #3 — `better-sqlite3` was not rebuilt for Electron (confirmed in logs)

The build log shows:
```
⚠  Could not rebuild better-sqlite3. The app may fail to start.
   node-gyp failed — Visual Studio 2017 or newer was not found
The native module on disk was compiled for regular Node.js, not for Electron's Node.js ABI. The app tried to use it anyway.

Fixes Required
1. Add node_modules to files and asarUnpack to electron-builder.yml:
yamlfiles:
  - dist/electron/main.cjs
  - dist/electron/preload.js
  - dist/electron/public/**/*
  - package.json
  - node_modules/**/*        # ← ADD THIS

asarUnpack:                  # ← ADD THIS SECTION
  - node_modules/better-sqlite3/**/*
  - node_modules/bufferutil/**/*
  - node_modules/utf-8-validate/**/*
2. Do the same in the ebConfig object inside script/build.electron.ts (it overrides electron-builder.yml at runtime).
3. Fix the native module rebuild. Install Visual Studio Build Tools on the Windows machine:

Download from: https://visualstudio.microsoft.com/visual-cpp-build-tools/
Select "Desktop development with C++" workload
Then before running the build, run: npx electron-rebuild -f -w better-sqlite3
